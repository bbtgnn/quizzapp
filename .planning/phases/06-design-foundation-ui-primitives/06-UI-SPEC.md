# Phase 06 — UI Design Contract

> Visual and interaction contract for **Design foundation & UI primitives** (TOK-01, CMP-01). Generated by gsd-ui-researcher, verified by gsd-ui-checker.  
> **Stack:** SvelteKit + Tailwind CSS v4 (`src/routes/layout.css`). **No shadcn registry** — custom primitives under `$lib`.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (custom tokens + Svelte components) |
| Preset | `tailwind-v4-custom` — tokens via Tailwind v4 `@theme` / CSS variables in `layout.css` (greenfield extension) |
| Component library | none (project-owned primitives: `Button`, `Panel`, run layout helpers in later phases) |
| Icon library | none required this phase; if icons added later, prefer inline SVG or a single small set — document choice in token docs |
| Font | **Display + Title:** one rounded, friendly webfont (e.g. **Nunito** Variable or **Quicksand** — final family at executor discretion per **D-06**, must be documented). **Body + Label:** system UI stack (`ui-sans-serif`, `system-ui`, `sans-serif`) per **D-04** |

**TOK-01 traceability:** All color roles, radii, shadows, spacing, and typography roles below are implemented as **design tokens** exposed to Tailwind utilities from `src/routes/layout.css` (and documented briefly per roadmap success criteria).

**CMP-01 traceability:** Primary actions use **large, high-visibility** styles; **minimum touch target** and **weight** documented in § Button variants; secondary/tertiary are **visually distinct** from primary per **D-07**.

## Layer Model: Stage vs Card vs Answers

| Layer | Z-order (relative) | Visual contract | Locked by |
|-------|-------------------|-----------------|-----------|
| **Stage** | Back | **Dark “broadcast” background** filling the viewport; may include **animated CSS motif** (e.g. sliding grid). **Light-colored chrome/labels** may sit directly on the stage when not on a card. | **D-01**, **D-02**, **D-13** |
| **Stage overlay / content column** | Middle | Max-width column, vertical rhythm; houses floating panels and answer rails. | ROADMAP funnel/run (this phase: style-lab only) |
| **Question / content card** | Front of column | **White or near-white surface**, **dark text** inside the card for primary reading; **noticeable radius + shadow** (depth, not flat banding). | **D-03**, **D-10** |
| **Answer actions** | Same column, **outside** the white card | **Not inside** the card. **Each option has a distinct hue** (“buzzer” read). Layout: horizontal wrap or vertical stack per breakpoint; **when a step has no answer buttons**, omit the rail and keep the **card centered** as the focal element. | **D-15** |

**Reduced motion (stage):** When `prefers-reduced-motion: reduce`, stage animation is **static or minimally animated** — no busy looping grid motion (**D-13**, **D-18**).

## Spacing Scale

Declared scale (multiples of 4px at **1rem = root**). **D-14:** `html { font-size: 112.5%; }` — rem-based utilities scale uniformly; document in token docs.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`0.25rem` before root scale; use theme spacing token) | Icon gaps, tight inline padding |
| sm | 8px | Compact spacing, label gaps |
| md | 16px | Default padding inside buttons/panels |
| lg | 24px | Section padding, card padding |
| xl | 32px | Layout gaps between major blocks |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level vertical rhythm |

**Exceptions:** **Minimum 44×44px touch target** for primary (and destructive when primary-sized) controls — may exceed token padding via min-height/min-width (**D-09**, **CMP-01**). Icon-only buttons must meet the same minimum and **must expose an accessible name** (`aria-label`, `title` only if no better option, or visually hidden text) — never icon-only with no name.

## Typography

**Weights (exactly two):** `400` (regular), `600` (semibold). No other weights in the quiz-show system unless a future phase amends this spec.

**Line height:** Body/label **1.5**; display/title **1.2** (recommended).

| Role | Size (css rem) | Weight | Line height | Usage |
|------|----------------|--------|-------------|--------|
| Display | `clamp(2rem, 4vw, 2.75rem)` | 600 | 1.2 | Hero / show title on funnel & run |
| Title | `1.5rem` | 600 | 1.2 | Card titles, section headings |
| Body | `1rem` | 400 | 1.5 | Paragraphs inside white cards |
| Label | `0.875rem` | 600 | 1.5 | Uppercase or sentence labels on stage, form labels |

**D-05:** Display/title use the **rounded, friendly** webfont. **D-04:** Body/label use **system UI** stack.

## Color

**Composition (60 / 30 / 10):**

| Role | Example token | Example hex | Usage (area of screen / element class) |
|------|----------------|-------------|----------------------------------------|
| **Dominant (~60%)** | `--color-stage` | `#0c0c12` | Stage background; largest visible area |
| **Secondary (~30%)** | `--color-surface-elevated`, `--color-card` | `#f7f7f8` (chrome strips), `#ffffff` (question card) | Floating panels, cards, nav strips — readable islands on the stage |
| **Accent (~10%)** | `--color-primary` | `#e11d48` (example; executor may tune) | **Reserved for:** default **primary CTA** fill/outline, **link-on-stage** emphasis, **keyboard focus ring** on in-scope interactive chrome — **not** for body text inside cards |
| **Destructive** | `--color-destructive` | `#dc2626` | Destructive actions only (**D-08**) |

**Light on dark:** Chrome, metadata, and labels **on the stage** use **light foreground** on `--color-stage` (e.g. `#f4f4f5`) per **D-02**.

**Dark on light:** **Inside** the white/near-white card, body and headings use **dark** foreground (e.g. `#18181b`) per **D-03**.

### Per-answer color slots (multicolor buzzers)

Minimum **four** distinct answer surface tokens for MCQ-style steps; support **N ≥ 4** via cycling or extension in implementation.

| Slot | Token | Example hex | Notes |
|------|-------|-------------|--------|
| Answer A | `--color-answer-a` | `#2563eb` | Saturated, mutually distinct |
| Answer B | `--color-answer-b` | `#16a34a` | |
| Answer C | `--color-answer-c` | `#ca8a04` | |
| Answer D | `--color-answer-d` | `#9333ea` | |
| Answer E+ | `--color-answer-n` | cycle A–D or extend palette | Document contrast vs white/light text choice |

Each answer button: **default**, **hover**, **active/disabled**, and **feedback** (success/wrong) states must be defined in tokens or component variants (contrast-safe).

## Button Variants (CMP-01 + D-07–D-09)

| Variant | Shape | Visual hierarchy | Typical use |
|---------|-------|------------------|-------------|
| **Primary** | **Pill** (radius ≥ height/2) | Filled with `--color-primary` (accent); **semibold** label; **min height 44px**, **min width** ≥ 44px or horizontal padding ≥ 24px so hit area meets **D-09** | Main CTAs on funnel/run |
| **Secondary** | **More rectangular** (smaller radius than primary, e.g. `md` token) | Outlined or muted fill; clearly **less prominent** than primary | Alternate actions |
| **Tertiary** | **Rectangular** (may match secondary radius family) | Text-only or ghost; **lowest** emphasis | Cancel, back, low-risk |
| **Destructive** | Match secondary/tertiary geometry unless it is the **primary** action of a step | Uses `--color-destructive`; if sized as main action, use **primary-sized** 44px target | Delete, irreversible |

**Focus:** Always show **visible `:focus-visible` ring** (e.g. 2px ring in accent or high-contrast) — **D-16** requires keyboard usability even when custom cursor is present.

## Cursor (pointer devices)

| Rule | Detail |
|------|--------|
| **Where** | **In-scope** quiz-show surfaces (funnel + run when implemented; this phase: style-lab page) may use a **large custom cursor** graphic |
| **Fallback** | **`pointer: coarse` (touch):** no custom cursor — system default |
| **Hotspot** | Executor chooses; must not obscure click targets |
| **Accessibility** | Custom cursor **does not replace** focus indicators; **D-16** |

## Motion & Accessibility

| Concern | Contract |
|---------|----------|
| **Stage background** | CSS-based looping motion allowed; **reduced motion** = static or minimal motion (**D-13**) |
| **Transitions** | Prefer **CSS**, **Svelte transitions**, or **Web Animations API** |
| **Optional library** | **anime.js** only if sequencing/effects are impractical in CSS — justify in plan (bundle, lazy load) (**D-18**) |
| **`prefers-reduced-motion: reduce`** | **Confetti / shake / large motion disabled**; substitute **instant or minimal** feedback (e.g. short color flash, static icon, immediate state change) (**D-18**) |

---

## Answer Feedback FX (run + style-lab demo)

| Outcome | Expressive (full motion) | Reduced motion |
|---------|---------------------------|----------------|
| **Positive / correct** | Celebratory burst (e.g. **confetti** or particle burst) | Static success icon or brief color pulse on the answer control — **no** particle systems |
| **Negative / wrong** | Clear wrong affordance (e.g. **card shake** or answer wobble) | **No** shake; use color/state change only |

Same **class** of feedback (success vs error) must be **instantly readable** (**D-17**). Examples are **non-exhaustive**; implementations may swap equivalents.
