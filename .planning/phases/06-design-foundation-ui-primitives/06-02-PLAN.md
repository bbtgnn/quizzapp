  <action>
    Create `src/lib/components/Button.svelte` (Svelte 5, `$props`, **native** `<button type="button">` — not `<div>`).

    **Public API:** `variant: 'primary' | 'secondary' | 'tertiary' | 'destructive'` (default `'primary'`), `class` merge, spread rest props, `children` snippet.

    **Styling (token-backed only — no raw hex in component):**
    - **Primary:** **pill** shape — `rounded-full` (or equivalent where height/2 ≤ radius); **filled** `bg-primary`, **semibold** label; **minimum** `min-h-[44px]` and `min-w-[44px]` **or** horizontal padding ≥ `24px` (`px-6` or `px-[24px]`) **plus** `min-h-[44px]` so hit area meets **D-09** / UI-SPEC Button table.
    - **Secondary:** **more rectangular** — use `rounded-(--radius-control)` or theme radius utility (e.g. `rounded-[var(--radius-control)]` / generated `rounded-control` if theme exposes it); **outlined or muted** fill; clearly less prominent than primary.
    - **Tertiary:** **ghost / text**; lowest emphasis; rectangular radius family matches secondary.
    - **Destructive:** uses **destructive** token (`bg-destructive` or outline pairing); **must not** look like primary — different hue/shape treatment per **D-08** / UI-SPEC.
    - **Focus:** all variants: `focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-offset-2` using **primary** or high-contrast ring color token (UI-SPEC: keyboard focus visible, **D-16**).
    - **Disabled:** `disabled:opacity-50 disabled:pointer-events-none` (or equivalent) — must remain **not** misleading as enabled.

    Export no side effects; keep classes compatible with **prettier-plugin-tailwindcss** (project standard).
  </action>
